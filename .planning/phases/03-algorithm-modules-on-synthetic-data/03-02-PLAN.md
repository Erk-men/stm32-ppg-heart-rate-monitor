      - IDLE: when filtered rises above s_threshold, transition to RISING and set
        s_peak_val = filtered.
      - RISING: while filtered keeps increasing, update s_peak_val = filtered. When filtered

        s_refractory_end = millis() + REFRACTORY_MS, then transition to REFRACTORY.
      - REFRACTORY: ignore all samples (this is what suppresses the dicrotic notch, SIG-03)
        until millis() >= s_refractory_end, then transition to FALLING.
      - FALLING: wait until filtered drops back below s_threshold, then transition to IDLE,
        ready for the next beat.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filtered value -> state machine | In-process value from the SIG-01 stage; trusted but drives all branching |
| millis() delta -> BPM divide | Unsigned subtraction feeds a division — the primary arithmetic hazard |
| computed bpm -> s_bpm_buf insertion | SIG-04 bounds check is the gate protecting the rolling-average buffer |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-06 | Denial of Service (divide by zero) | bpm = 60000UL / interval_ms | mitigate | interval_ms guarded `!= 0`; s_last_peak_ms==0 sentinel skips the first-beat computation entirely |
| T-03-07 | Tampering (buffer overflow) | s_bpm_buf[5] ring index | mitigate | s_bpm_idx advanced via `% 5`; s_bpm_count clamped at 5 — index can never exceed 4 |
| T-03-08 | Tampering (out-of-range read) | rolling average over s_bpm_count entries | mitigate | average loop bounded by s_bpm_count (0..5); s_bpm_count >= 1 guaranteed at the divide site |
| T-03-09 | Information disclosure (uninitialized state) | s_state, s_threshold, s_bpm_buf, s_bpm_count | mitigate | algorithm_init() explicitly resets every static; called from main.c before first algorithm_process |
| T-03-10 | Spoofing (false-positive beat) | dicrotic notch counted as a beat | mitigate | 350ms REFRACTORY state ignores all samples after a peak — SIG-03 suppression is the explicit defense |
| T-03-11 | Denial of Service (integer overflow) | rolling-average sum | accept | max sum = 5 * 200 = 1000, far within uint32_t — overflow impossible by the SIG-04 bound |
</threat_model>
