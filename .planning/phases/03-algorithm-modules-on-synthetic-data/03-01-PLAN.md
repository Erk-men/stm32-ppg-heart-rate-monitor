<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| sine_table -> algorithm_process | Synthetic samples are compile-time constants — fully trusted, but values must stay within declared uint16_t and ADC count range |
| millis() -> interval arithmetic | Unsigned 1ms counter; subtraction is the only operation crossing into BPM math |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering (memory safety) | s_ma_buf[32] circular index | mitigate | s_ma_idx advanced only via `(s_ma_idx + 1) & 0x1F` — mask guarantees index stays 0..31; no unbounded write |
| T-03-02 | Denial of Service (integer overflow) | s_ma_sum running sum | accept | uint32_t holds max 32 * 65535 = 2,097,120; synthetic samples capped ~2200 — overflow impossible by construction |
| T-03-03 | Denial of Service (divide by zero) | bpm = 60000UL / interval_ms | mitigate | interval_ms guarded `!= 0` before the divide; s_last_peak_ms==0 sentinel skips the first computation |
| T-03-04 | Information disclosure (uninitialized state) | s_ma_buf, s_prev_filtered, s_last_peak_ms | mitigate | algorithm_init() explicitly zeros every static before first algorithm_process call; main.c calls algorithm_init() after adc_init() |
| T-03-05 | Tampering (stack depth) | algorithm_process call chain | accept | No recursion; flat call into uart_write_* leaf functions; bare-metal stack budget unaffected |
</threat_model>
