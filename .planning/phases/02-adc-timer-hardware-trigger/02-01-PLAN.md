Create Src/tim3.c with the following implementation. All values come from RESEARCH.md Section 2.

Include only: "stm32f0xx.h" and "tim3.h".

void tim3_init(void) must perform these register writes in order:

1. Enable TIM3 clock: RCC->APB1ENR |= RCC_APB1ENR_TIM3EN;

2. Set prescaler: TIM3->PSC = 79;
   Rationale: 8,000,000 Hz / (79+1) = 100,000 Hz (100 kHz counter clock).

3. Set auto-reload: TIM3->ARR = 999;
   Rationale: 100,000 Hz / (999+1) = 100 Hz update event rate.

4. Set master mode to Update Event: TIM3->CR2 = TIM_CR2_MMS_1;
   Rationale: MMS[2:0] = 010 (bit 5 of CR2 = TIM_CR2_MMS_1 = 0x20). This routes the Update Event to the TRGO output that triggers ADC1. Do NOT enable TIM3 interrupts — TIM3 runs autonomously.

5. Enable counter: TIM3->CR1 |= TIM_CR1_CEN;

Do NOT write TIM3->EGR = TIM_EGR_UG. Letting the first Update Event happen naturally avoids a spurious TRGO pulse at startup (see RESEARCH.md Section 2, "No EGR.UG Needed").

Define the two globals at file scope:

  volatile uint8_t  g_adc_ready  = 0;

Implement void adc_init(void) with this EXACT register sequence (order is critical — see RESEARCH.md Pitfalls 2, 4, 7):

Step 1 — PA0 analog mode (GPIOA clock is already enabled by usart2_init, no duplicate needed):
  GPIOA->MODER |= GPIO_MODER_MODER0;
  // Sets MODER[1:0] = 11 (analog mode). PUPDR default 00 is already correct.

Step 2 — Enable ADC1 clock on APB2:
  RCC->APB2ENR |= RCC_APB2ENR_ADCEN;

Step 3 — Start HSI14 dedicated ADC oscillator and wait for it to stabilize:
  RCC->CR2 |= RCC_CR2_HSI14ON;
  while (!(RCC->CR2 & RCC_CR2_HSI14RDY));

Step 4 — Calibrate ADC (ADEN must be 0 here — never set ADCAL with ADEN=1):
  ADC1->CR |= ADC_CR_ADCAL;
  while (ADC1->CR & ADC_CR_ADCAL);
  // Self-clearing when calibration completes (~6 µs at 14 MHz HSI14).

Step 5 — Configure CFGR1 before enabling ADC (write lock applies after ADEN=1):
  ADC1->CFGR1 = ADC_CFGR1_EXTSEL_0 | ADC_CFGR1_EXTSEL_1  /* EXTSEL=011 → TIM3_TRGO */
              | ADC_CFGR1_EXTEN_0                           /* EXTEN=01  → rising edge */
              | ADC_CFGR1_OVRMOD;                           /* overwrite DR on overrun */
  // ADC_CFGR1_EXTSEL_0 | ADC_CFGR1_EXTSEL_1 = 0x000000C0 (bits [8:6] = 011)
  // ADC_CFGR1_EXTEN_0 = 0x00000400 (bits [11:10] = 01)
  // ADC_CFGR1_OVRMOD = bit 12 — new value overwrites old if consumer misses a sample.
  // Leave RES=00 (default) for 12-bit resolution.
  // Leave CFGR2 at reset value 0x00 so CKMODE=00 (HSI14 async) is selected.

Step 6 — Set sample time to maximum (SMP=111 = 239.5 cycles) for best SNR:
  ADC1->SMPR = ADC_SMPR_SMP;   // ADC_SMPR_SMP = 0x7 (all three SMP bits)

Step 7 — Select channel 0 (PA0 = ADC_IN0):
  ADC1->CHSELR = ADC_CHSELR_CHSEL0;   // = 0x00000001

Step 8 — Enable EOC interrupt:
  ADC1->IER = ADC_IER_EOCIE;

Step 9 — Set NVIC priority for ADC1_IRQn (priority 1, below SysTick at 0):
  NVIC_SetPriority(ADC1_IRQn, 1);
  NVIC_EnableIRQ(ADC1_IRQn);

Step 10 — Enable ADC and wait for voltage reference to settle:
  ADC1->CR |= ADC_CR_ADEN;
  while (!(ADC1->ISR & ADC_ISR_ADRDY));

Step 11 — Arm triggered conversion mode:
  ADC1->CR |= ADC_CR_ADSTART;
  // With EXTEN != 00, ADSTART arms the ADC to wait for the next TIM3 TRGO rising edge.
  // It does NOT start an immediate conversion.

Then implement void ADC1_IRQHandler(void):
  void ADC1_IRQHandler(void)
  {
      if (ADC1->ISR & ADC_ISR_EOC) {
          g_adc_sample = (uint16_t)(ADC1->DR & 0x0FFF);  /* reading DR clears EOC */
          g_adc_ready  = 1;
      }
  }
  // Reading ADC1->DR clears the EOC flag automatically. No explicit w1c needed.
  // The 0x0FFF mask ensures only the 12-bit result is captured.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| analog input → ADC | PA0 accepts 0–3.3V from external LM358 circuit — values outside this range can damage the pin |
| ISR → main loop | g_adc_sample and g_adc_ready cross from interrupt context to thread context via volatile globals |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | ADC1->DR read in ISR | mitigate | Mask result with 0x0FFF to prevent bit-field bleed from reserved bits corrupting g_adc_sample |
| T-02-02 | Denial of Service | HSI14 startup hang | mitigate | Poll RCC_CR2_HSI14RDY in a bounded spin loop; if hardware is healthy this resolves in ~1 µs — no timeout needed at this stage |
| T-02-03 | Denial of Service | ADCAL with ADEN=1 (undefined behavior) | mitigate | Initialization sequence sets ADCAL before any ADEN write — enforced by code order |
| T-02-04 | Denial of Service | EOC not cleared → runaway ISR | mitigate | Reading ADC1->DR clears EOC automatically on F0; ISR reads DR unconditionally |
| T-02-05 | Information Disclosure | g_adc_sample read without synchronization | accept | Both accesses are 16-bit aligned reads on M0 — atomic by architecture; no torn read possible |
| T-02-06 | Elevation of Privilege | External voltage > 3.3V on PA0 | accept | Hardware constraint: LM358 powered from 3.3V rail; voltages above VDDA require hardware protection not in scope here |
</threat_model>
