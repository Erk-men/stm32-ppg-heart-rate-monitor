    `Inc/systick.h` — header guard SYSTICK_H, include <stdint.h>, declare:

      - `uint32_t millis(void);`

    `Src/systick.c` — include "stm32f1xx.h" and "systick.h".
      - Define `static volatile uint32_t s_ticks = 0;` at file scope (D-03: not extern).
      - Implement `systick_init(void)`: call `SysTick_Config(71999)` (CMSIS helper from core_cm3.h — sets LOAD=71999, VAL=0, CTRL with CLKSOURCE=HCLK + TICKINT + ENABLE, registers SysTick_IRQn priority 15 by default; that is acceptable for this phase). Then call `NVIC_SetPriority(SysTick_IRQn, 0)` to raise to highest priority per CLAUDE.md §Interrupt Priority Plan.

      - Implement `void SysTick_Handler(void)`: `s_ticks++;` — body is exactly one statement (D-02: ISR lives in driver file).

    The LOAD value 71999 = (72_000_000 Hz / 1000 Hz) - 1 — reload counts from LOAD down to 0 inclusive, so 72000 cycles = 1ms at 72MHz HCLK.

    Do NOT modify Src/stm32f1xx_it.c — the previous task ensured SysTick_Handler is not defined there. If the linker errors with "multiple definition of SysTick_Handler", delete the duplicate from stm32f1xx_it.c, not from systick.c.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | D (Denial of Service) | SysTick ISR | accept | ISR body is a single increment — no busy work, no allocation, runs in ~6 cycles. Cannot self-DoS. |
| T-01-02 | I (Information Disclosure) | s_ticks counter | mitigate | Counter is `static` (file-local) and not exposed via extern (D-03). Only accessible via millis() — read-only API. |
| T-01-03 | T (Tampering) | uint32_t millis() rollover | accept | Rollover at 2^32 ms = ~49.7 days is benign — all consumers (Phase 2–4) MUST use unsigned subtraction `(now - last)` which is rollover-safe per C standard for unsigned arithmetic. Documented in SKELETON.md. |
</threat_model>
