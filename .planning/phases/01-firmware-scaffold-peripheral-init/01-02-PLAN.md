  <action>
    Create Inc/usart.h and Src/usart.c per D-01 (driver pair) and D-04 (two-primitive API, no stdlib).

    `Inc/usart.h`: header guards USART_H, include `<stdint.h>`, declare the three functions in the interfaces block above. No other exports.

    `Src/usart.c`: include `"stm32f1xx.h"` and `"usart.h"`. Implement:

    **usart2_init()** in this exact order (RCC gotcha: clocks before any peripheral register write):
    1. Enable GPIOA clock: `RCC->APB2ENR |= RCC_APB2ENR_IOPAEN;` (GPIOA is on APB2 — F103 gotcha)
    2. Enable USART2 clock: `RCC->APB1ENR |= RCC_APB1ENR_USART2EN;` (USART2 is on APB1, PCLK1=36MHz)
    3. Configure PA2 (TX): clear `GPIOA->CRL` bits [11:8] then set to 0xB (CNF=10 AF push-pull, MODE=11 50MHz). Use mask `GPIO_CRL_CNF2 | GPIO_CRL_MODE2` for clearing and `(0x0B << 8)` for setting.
    4. Configure PA3 (RX): clear `GPIOA->CRL` bits [15:12] then set to 0x4 (CNF=01 floating input, MODE=00 input). Use mask `GPIO_CRL_CNF3 | GPIO_CRL_MODE3` for clearing and `(0x04 << 12)` for setting.
    5. Set baud rate: `USART2->BRR = 0x139;` — exact value 0x139 (313 decimal) for 115200 at 36MHz PCLK1 per CLAUDE.md §USART2. Do NOT compute at runtime; the magic number is documented.
    6. Enable USART2 with TX enabled, 8N1 (defaults): `USART2->CR1 = USART_CR1_UE | USART_CR1_TE;` (UE bit 13, TE bit 3). No interrupts. No RX enable required for Phase 1 (RX is wired but unused).

    **uart_write_str(const char *s)**:
    - If `s == NULL` return immediately.
    - Loop: while `*s` is non-zero, poll `while (!(USART2->SR & USART_SR_TXE));` then `USART2->DR = (uint8_t)*s++;`.

    **uart_write_u32(uint32_t n)**:
    - Special-case 0: emit '0' and return.
    - Otherwise build digits into a small fixed buffer (max 10 chars for 2^32-1). Extract digits via `n % 10` + `n /= 10` loop (compiler emits umlal/udiv — acceptable on Cortex-M3; F103 has hardware divide). Write digits in reverse order using a `char buf[10]; int i = 0;` then emit `buf[i-1] ... buf[0]` via the same TXE-poll-then-DR pattern as uart_write_str. Do NOT allocate. Do NOT call snprintf or itoa.

    Do NOT enable USART2 RX, USART2 interrupts, or DMA. Do NOT poll TC (TC poll is optional — if added, do it once after the last byte, not per-byte, to avoid 87µs/byte overhead vs 86.8µs/byte ideal).

    The full register sequence and bit positions are in CLAUDE.md §GPIO Configuration and §USART2. Do not invent values.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| MCU → host UART | One-way TX only. RX line (PA3) is configured as floating input but never read. No data flows into firmware from outside. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04 | T (Tampering) | uart_write_str NULL pointer | mitigate | usart.c explicitly checks `if (s == NULL) return;` at the top of uart_write_str — bare-metal nullptr deref reboots Cortex-M3 via hardfault, so this is a robustness fix. |
| T-01-05 | D (Denial of Service) | uart_write_str blocking on TXE | accept | Polling TX blocks the main loop for ~87µs/byte at 115200. A 15-char line takes ~1.3ms — far less than the 1000ms print interval. Cannot stall the system in Phase 1. Phase 2 (100Hz ADC ISR) is also unaffected because main-loop polling does not mask interrupts. |
| T-01-06 | I (Information Disclosure) | UART TX over USB virtual COM | accept | Local debug interface only. Course context — no PII, no credentials. ASVS L1 does not apply to debug serial output on student hardware. |
| T-01-07 | D (DoS) | uart_write_u32 stack buffer | mitigate | Fixed-size `char buf[10]` covers maximum 10-digit uint32_t (4294967295). No dynamic allocation. No bounds risk. |
</threat_model>
