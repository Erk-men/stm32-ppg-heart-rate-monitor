    The block declares four locals: `uint16_t cal_min = 4095;`,
    `uint16_t cal_max = 0;`, `uint32_t cal_count = 0;`, `uint32_t cal_start = millis();`.
    It prints "CALIBRATION_MODE: sampling 5s...\r\n" via uart_write_str. It then runs
    `while ((millis() - cal_start) < 5000UL)` and, on each `g_adc_ready`, clears the
    flag, copies `g_adc_sample` into a local `uint16_t s`, updates `cal_min`/`cal_max`,
    and increments `cal_count`. After the window it computes
    `uint16_t cal_amp = (cal_max > cal_min) ? (uint16_t)(cal_max - cal_min) : 0;` and
    prints one line in the exact format: `CAL min=` then cal_min, ` max=` then
    cal_max, ` amp=` then cal_amp, ` n=` then cal_count, then `\r\n`. Use only
    uart_write_str for literals and uart_write_u32 for numeric values.

    Constraints (all from PATTERNS.md / CLAUDE.md):
    - Integer arithmetic only. No float, no double. The `cal_max - cal_min`
      subtraction is guarded by the ternary so it can never underflow.
    - Copy `g_adc_sample` (declared `volatile uint16_t`) into a local before use —
      avoids a volatile re-read race on Cortex-M0.
    - The block MUST sit AFTER `tim3_init()` and `adc_init()` so TIM3 TRGO is firing
      and `g_adc_ready` actually sets (RESEARCH.md Pitfall 1 — running before ADC
      init reports n=0/amp=0).
    - Enclose the block body in a brace pair `{ ... }` so the locals are scoped, as
      shown in Pattern 1D.

    Leave `#define CALIBRATION_MODE` commented out in production (it is uncommented

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LM358 analog -> ADC1 (PA0) | Real-world analog voltage crosses into the firmware; magnitude and noise are uncontrolled |
| ADC ISR -> main loop | g_adc_sample / g_adc_ready shared between the ADC interrupt and the main loop |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | g_adc_sample volatile read in CALIBRATION_MODE loop | mitigate | Copy `g_adc_sample` into a local `uint16_t s` immediately after clearing `g_adc_ready`, before any comparison — prevents a torn/re-read value on Cortex-M0 |
| T-04-02 | Denial of Service | CALIBRATION_MODE 5s busy-wait loop | accept | Intentional 5-second blocking window before the main loop; bounded by `millis()` elapsed check; runs only when the define is uncommented |
| T-04-03 | Information disclosure | Serial output of raw ADC values | accept | Embedded lab device, no network, no sensitive data; CAL output is intended report evidence |
| T-04-04 | Denial of Service | ADC starvation if tim3_init/adc_init left gated after SYNTHETIC_TEST removal | mitigate | Task 1 makes both init calls unconditional; Task 3 hardware check confirms cal_count n is ~500 (non-zero proves TRGO is firing) |
</threat_model>
